# TeamBridge

TeamBridge — платформа для командной работы, объединяющая чат и управление задачами в едином интерфейсе. Приложение позволяет эффективно обсуждать идеи в реальном времени и мгновенно преобразовывать сообщения в задачи с контролем исполнения.

## Обзор

TeamBridge создан для команд, которым важно сохранять связь между коммуникацией и действиями. Вместо переключения между мессенджерами и системами управления задачами, пользователи работают в одном пространстве: обсуждают вопросы в чате, выделяют ключевые моменты и превращают их в задачи с назначением, дедлайнами и приоритетами.

Платформа поддерживает гибкую организацию работы через проекты и организации, обеспечивает безопасность данных и предоставляет интуитивный интерфейс для повседневного использования.

## Функциональность

### Чат и коммуникация
Пользователи обмениваются сообщениями в проектных чатах с мгновенной синхронизацией. Сообщения можно быстро превратить в задачу с помощью контекстного меню (правый клик → "Сделать задачей"). Поддерживаются вложения файлов и медиа, а также уведомления о новых сообщениях и активностях.

### Управление задачами
Задачи отображаются в виде Kanban-доски с несколькими видами фильтрации. Дополнительно доступен календарный вид для планирования сроков. Каждая задача поддерживает детализацию: приоритет, исполнитель, дедлайн, комментарии. Реализованы фильтрация и поиск по задачам.

### Организации и проекты
Работа в TeamBridge организована через иерархию: организации содержат проекты, к которым подключаются участники. Предусмотрена ролевая модель доступа — владелец, администратор и участник — с соответствующими правами. Команды управляются через приглашения и настройку прав.

### Безопасность и аутентификация
Аутентификация реализована через Firebase Auth с использованием JWT. Регистрация и вход подтверждаются по email, доступно восстановление пароля. Сообщения шифруются на стороне сервера. Ввод пользовательского контента проходит фильтрацию от XSS-атак с помощью DOMPurify.

## Архитектура

### Технологический стек
Frontend построен на React 19 с TypeScript и сборкой через Vite. UI реализован на кастомных компонентах без использования внешних библиотек. Управление состоянием осуществляется через Context API (AuthContext, OrganizationContext, ProjectContext). В качестве бэкенда используется Firebase — Authentication для авторизации, Firestore и Realtime Database для хранения и синхронизации данных. Деплой выполняется на Render.
