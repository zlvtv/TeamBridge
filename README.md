# TeamBridge

TeamBridge — корпоративный мессенджер с интегрированной системой управления задачами. Приложение объединяет организации, проекты, роли, проектный чат, задачи, уведомления и рабочую координацию в одном интерфейсе.

## Что это за проект

TeamBridge построен вокруг идеи, что командная коммуникация и исполнение не должны жить в разных сервисах. Внутри одного рабочего пространства команда может:

- общаться в проектных чатах
- тегать пользователей и роли через `@mention`
- создавать организации и проекты с отдельным составом участников
- назначать системные, кастомные и локальные роли
- превращать сообщения в задачи
- вести задачи в списке, kanban и календаре
- получать внутренние и браузерные уведомления

Проект особенно подходит для сценариев, где обсуждение должно быстро превращаться в исполнимую задачу без потери контекста.

## Актуальная функциональность

### Организации и проекты

- Иерархия из организаций и проектов.
- Автоматически создаваемый общий проект внутри организации.
- Приглашения в организацию по ссылке.
- Разделение прав на уровне организации и проекта.
- Поддержка куратора проекта.

### Роли и структура команды

- Системные роли организации: `owner`, `admin`, `member`.
- Кастомные роли организации с цветами.
- Локальные роли проекта, которые действуют только внутри конкретного проекта.
- Назначение ролей участникам и управление ими через интерфейс модалок.
- Упоминания пользователей и ролей в сообщениях.

### Чат и коммуникация

- Проектный чат в реальном времени.
- Ветки обсуждений.
- Эмодзи, вложения и опросы.
- Отложенная отправка сообщений.
- Системные сообщения по важным событиям проекта.
- Внутреннее обсуждение внутри задачи.

### Задачи и task-board

- Создание задачи напрямую из чата.
- Название, описание, исполнители, приоритет, статус, теги.
- Срок задачи с датой и точным временем.
- Напоминания по задаче для исполнителей.
- Внутренний чат задачи.
- Представления: список, kanban, календарь.
- Drag-and-drop изменения статуса в kanban.
- Вкладки задач проекта, организации и всех задач пользователя.
- Фильтрация, поиск, сортировка и фильтр по тегам.

### Уведомления и присутствие

- Внутренний центр уведомлений.
- Браузерные уведомления при разрешении со стороны пользователя.
- Уведомления по упоминаниям, ответам и событиям задач.
- Напоминания по задачам для ответственных.
- `last seen` / статус недавней активности участников.

### Профиль и доступ

- Регистрация, вход, подтверждение email и восстановление пароля.
- Редактирование профиля: никнейм, полное имя, описание, аватар.
- Проверка уникальности username.
- Удаление профиля и обработка удаленных пользователей в интерфейсе.

## Архитектура на уровне приложения

Основные доменные слои разнесены по контекстам:

- `AuthContext` — аутентификация, профиль пользователя, username, last seen
- `OrganizationContext` — организации, состав участников, приглашения, роли организации
- `ProjectContext` — проекты, текущий проект, участники проектов, unread и синхронизация метаданных

Ключевая логика работы с данными вынесена в сервисы:

- `organizationService`
- `projectService`
- `taskService`
- `messageService`
- `notificationService`
